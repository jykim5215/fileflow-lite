# Security Policy

## 지원 버전

최신 GitHub Release만 보안 수정 대상으로 지원합니다.

## 보안 원칙

- FileFlow Lite는 파일을 외부 서비스로 전송하지 않습니다.
- 업데이트 확인 외에는 네트워크를 사용하지 않습니다.
- 시스템·숨김 파일과 재분석 지점을 기본적으로 제외합니다.
- 기존 파일 자동 덮어쓰기를 허용하지 않습니다.
- 실행 전후 상태와 Undo 가능 여부를 로컬 JSON 저널에 기록합니다.
- 릴리스 다운로드는 SHA-256 체크섬을 확인합니다.
- v0.2 셸 DLL은 네트워크나 파일 스캔을 수행하지 않고 선택 항목 전달과 작업 프로세스 실행만 담당합니다.
- 관리형 파일 엔진은 COM Surrogate/Explorer 프로세스 밖에서 실행됩니다.
- v0.2 MSIX와 업데이트는 동일한 코드서명 인증서로 서명합니다.

## 취약점 신고

공개 이슈에 개인정보나 실제 로컬 경로를 올리지 마세요. GitHub 저장소의 Security 탭에서 비공개 취약점 신고를 사용하거나, 저장소 소유자에게 비공개로 연락해 주세요.

신고에는 재현 단계, 영향 범위, 사용 버전만 포함하고 개인 파일은 첨부하지 마세요.

## 알려진 경계

- 체크섬은 전송 오류와 비의도적 변조를 탐지하지만 코드 서명을 대체하지 않습니다.
- 네트워크 드라이브와 동기화 폴더는 다른 프로세스의 동시 수정 때문에 로컬 NTFS보다 보장이 약할 수 있습니다.
- 이 도구는 백업 프로그램이 아니므로 중요한 파일은 별도 백업을 유지하세요.
